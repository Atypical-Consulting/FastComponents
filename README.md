## Motivation

In 2008, Roy Fielding asserted, “REST APIs must be hypertext-driven,” hinting at a more integrated, efficient web. FastComponents embodies this vision, bringing the essence of RESTful architecture to .NET 8 through HTML-driven APIs.

With HTMX as its foundation, FastComponents revolutionizes web interactions. HTMX is a lightweight, dependency-free JavaScript library by Carson Gross, which leverages HTML as both the message and state container. It transcends traditional JSON limitations, allowing dynamic, form-shifting responses aligned with the principles of ReST (Representational State Transfer).

Then, there's FastEndpoints. It's a .NET library that streamlines endpoint creation, known for its remarkable performance benchmarks. Whether returning JSON, XML, or HTML fragments, FastEndpoints excels, fully harnessing HTMX's power, including WebSocket and SSE extensions for enhanced interactivity.

The server-side load is minimal due to the compact size of the responses. Furthermore, HTML fragments can be secured, preventing unintended client-side rendering. These fragments, generated by endpoints with or without parameters, are the core of FastComponents.

Infinite possibilities await with FastComponents – a platform where .NET efficiency meets modern web dynamics. Ready to dive in?

Build dynamic server-side applications in ASP.NET Core using HTMX and FastComponents, a modern approach to web development that simplifies back-end logic. FastComponents enables easy creation of MRAs, serving multiple resources effectively.

- Simplified server-side logic with .NET efficiency.
- Reduced client-side complexity, minimizing JavaScript reliance.
- HTMX enhances user experience with AJAX-like capabilities without full page reloads.

Downsides:
- Limited to server-side dynamics.
- Requires understanding of HTMX for optimal use.
